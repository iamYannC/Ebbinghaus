## trials.csv — Phase 1: Stimulus Creation

**Generated by:** `generate_design()` or custom construction  
**Used by:** `render_stimuli()`, `strip_answer_from_images()`, `evaluate_all()`, `analyze_results()`  
**Rows:** One per trial (e.g., 20 trials = 20 rows)

| # | Variable | Type | Values/Range | Role | How to Modify |
|---|----------|------|--------------|------|---------------|
| 1 | `trial_id` | integer | Auto-increment (1, 2, 3, ...) | Primary key | Auto-assigned by `generate_design()`. Don't edit manually unless constructing custom trials. |
| 2 | `n_comparisons` | integer | `2` (fixed) | Design | Number of test stimuli presented per trial. Fixed at 2 (the classic pairwise comparison). Reserved for future multi-way comparisons. |
| 3 | `orientation` | character | `"horizontal"`, `"vertical"`, `"diagonal"` | Layout | Controls spatial arrangement. Horizontal = left/right, vertical = top/bottom, diagonal = upper-left/lower-right. Edit in `generate_trial()` or filter existing trials. |
| 4 | `canvas_width` | integer | `{512, 768, 1024}` | Layout | Image width in pixels. Modify `CANVAS_SIZES` in `config/defaults.R`. |
| 5 | `canvas_height` | integer | `{512, 768, 1024}` | Layout | Image height in pixels. Modify `CANVAS_SIZES` in `config/defaults.R`. |
| 6 | `background_color` | character | `"#FFFFFF"`, `"#000000"` | Layout | Canvas background (black or white). Modify `BACKGROUND_POOL` in `config/defaults.R`. |
| 7 | `test_a_shape` | character | `"circle"`, `"square"` | Test stimulus | Shape of test stimulus A (position = left/top/upper-left). Modify `SHAPE_POOL` in `config/defaults.R`. |
| 8 | `test_a_size` | numeric | `(0, ∞)` | Test stimulus | Size in plot units (radius for circle, half-side for square). Computed as `proportion × min(canvas_width, canvas_height)`, where the proportion is drawn from `TEST_SIZE_RANGE` (default `[0.03, 0.08]`). With current defaults and canvas sizes, typical values fall roughly in `[15, 82]`. Modify `TEST_SIZE_RANGE` in `config/defaults.R`. |
| 9 | `test_a_color` | character | Hex color (e.g., `"#4682B4"`) | Test stimulus | Border color. Modify `COLOR_POOL` in `config/defaults.R`. |
| 10 | `test_a_fill` | character | Hex color or `NA` | Test stimulus | Fill color (`NA` = outline only). Modify `FILL_POOL` in `config/defaults.R`. |
| 11 | `test_b_shape` | character | `"circle"`, `"square"` | Test stimulus | Shape of test stimulus B (position = right/bottom/lower-right). |
| 12 | `test_b_size` | numeric | `(0, ∞)` | Test stimulus | Size in plot units. Same scale as `test_a_size` (see above). **This is the key comparison variable**—manipulated to create different tiers. |
| 13 | `test_b_color` | character | Hex color | Test stimulus | Border color. |
| 14 | `test_b_fill` | character | Hex color or `NA` | Test stimulus | Fill color. |
| 15 | `surround_a_shape` | character | `"circle"`, `"square"`, `NA` | Context stimulus | Shape of surround stimuli around A. `NA` when `surround_a_n == 0`. |
| 16 | `surround_a_size` | numeric | `(0, ∞)` or `NA` | Context stimulus | Size of each surround in plot units. Computed as `proportion × min(canvas_width, canvas_height)`, where the proportion is drawn from `SURROUND_SIZE_RANGE` (default `[0.02, 0.15]`). **Key illusion variable**—larger surrounds make test appear smaller. `NA` when `surround_a_n == 0`. Modify `SURROUND_SIZE_RANGE` in `config/defaults.R`. |
| 17 | `surround_a_n` | integer | `{0, 4, 5, 6, 7, 8}` | Context stimulus | Number of surrounds. `0` = Tier 0 (sanity check). Modify `SURROUND_N_RANGE` in `config/defaults.R`. |
| 18 | `surround_a_color` | character | Hex color or `NA` | Context stimulus | Border color of surrounds. |
| 19 | `surround_a_fill` | character | Hex color or `NA` | Context stimulus | Fill color of surrounds. |
| 20 | `surround_a_distance` | numeric | `(0, ∞)` or `NA` | Context stimulus | Distance from test center to surround centers. Controls crowding. Modify `SURROUND_DISTANCE_RANGE` in `config/defaults.R`. |
| 21 | `surround_b_shape` | character | `"circle"`, `"square"`, `NA` | Context stimulus | Shape of surround stimuli around B. |
| 22 | `surround_b_size` | numeric | `(0, ∞)` or `NA` | Context stimulus | Size of each surround around B. Same scale as `surround_a_size` (see above). **Manipulated relative to `surround_a_size` to create tiers**. |
| 23 | `surround_b_n` | integer | `{0, 4, 5, 6, 7, 8}` | Context stimulus | Number of surrounds around B. |
| 24 | `surround_b_color` | character | Hex color or `NA` | Context stimulus | Border color. |
| 25 | `surround_b_fill` | character | Hex color or `NA` | Context stimulus | Fill color. |
| 26 | `surround_b_distance` | numeric | `(0, ∞)` or `NA` | Context stimulus | Distance from test B center to surround centers. |
| 27 | `true_larger` | character | `"equal"`, `"a"`, `"b"` | Ground truth | **Computed by `verify_trial()`**—never set manually. Determined purely from `test_a_size` vs `test_b_size`. |
| 28 | `true_diff_ratio` | numeric | `[0, 1)` | Ground truth | **Computed by `verify_trial()`**. Formula: `abs(a - b) / max(a, b)`. A decimal proportion (e.g., 0.15 means 15% difference). |
| 29 | `tier` | integer | `{0, 1, 2, 3}` or `NA` | Ground truth | **Computed by `classify_tier()`**. Tier 0 = no surrounds, Tier 1 = equal sizes, Tier 2 = incongruent, Tier 3 = congruent. See SPECIFICATION.md §1.4. |
| 30 | `seed` | integer | Any integer or `NA` | Reproducibility | RNG seed used by `generate_trial()`. `NA` if the trial was manually constructed. Allows exact reproduction. Note: the `tier` parameter in `generate_trial()` accepts `NULL` (unconstrained) or `0`/`1`/`2`/`3` (targeted); `NA` is not a valid target tier—it is only an output of `classify_tier()`. |
| 31 | `file_format` | character | `"png"`, `"svg"`, `"webp"` | Rendering | Image format. Modify `FILE_FORMAT_POOL` in `config/defaults.R`. |
| 32 | `file_path` | character | Relative path, e.g., `"images/1_b_t3.png"` | Rendering | Path to rendered image. Auto-generated by `generate_design()` or `render_stimuli()`. Format: `<id>_<true_larger>_t<tier>.<ext>`. |

### How to modify trials.csv:

**Option 1: Generate a new design**
```r
source("R/generate_design.R")
trials <- generate_design(seed = 999, n_per_tier = 50)  # 50 trials per tier
# optional: write.csv(trials, "data/trials.csv", row.names = FALSE)
render_stimuli(trials)  # renders to images/
```

**Option 2: Filter existing trials**
```r
trials <- read.csv("data/trials.csv")
my_subset <- trials |> filter(tier == 1, orientation == "horizontal")
render_stimuli(my_subset)  # renders to images/
```

> **Tip:** If you want to change the default parameter pools or ranges (e.g., add
> a new shape, widen the size range), edit `config/defaults.R` before regenerating.
> For example:
> ```r
> # In config/defaults.R, change:
> SHAPE_POOL <- c("circle", "square", "triangle")  # Add triangles
> # Then regenerate
> ```

### Combinatoric space (discrete parameters only)

Using the current defaults in `config/defaults.R`, the number of distinct discrete combinations is:

| Parameter | Levels | Values |
|-----------|--------|--------|
| `orientation` | 3 | horizontal, vertical, diagonal |
| `canvas_width` | 3 | 512, 768, 1024 |
| `canvas_height` | 3 | 512, 768, 1024 |
| `background_color` | 2 | white, black |
| `test_*_shape` (×2) | 2 | circle, square |
| `test_*_color` (×2) | 5 | 5 hex codes |
| `test_*_fill` (×2) | 6 | 5 hex codes + `NA` |
| `surround_*_shape` (×2) | 2 | circle, square |
| `surround_*_n` (×2) | 5 (+1 for `n=0`) | 4, 5, 6, 7, 8 (or 0 = no surrounds) |
| `surround_*_color` (×2) | 5 | 5 hex codes |
| `surround_*_fill` (×2) | 6 | 5 hex codes + `NA` |
| `file_format` | 3 | png, svg, webp |

**Total discrete combinations: ~52.8 billion**

> **Note:** Six parameters are continuous rather than discrete: `test_a_size`,
> `test_b_size`, `surround_a_size`, `surround_b_size`, `surround_a_distance`,
> and `surround_b_distance`. These are drawn from ranges defined in
> `config/defaults.R` and make the true parameter space effectively infinite.
> The count above reflects only the discrete axes.

## prompts.csv — Phase 2: Evaluation

**Created manually** or programmatically  
**Used by:** `evaluate_all()`, `analyze_results()`  
**Rows:** One per prompt variant (e.g., 3 prompts = 3 rows)

| # | Variable | Type | Values/Range | Role | How to Modify |
|---|----------|------|--------------|------|---------------|
| 1 | `prompt_id` | integer | Auto-increment (1, 2, 3, ...) | Primary key | Assign sequentially. Used to join with `evals.csv`. |
| 2 | `system_prompt` | character | Free text | Prompt content | Sets the model's persona/behavior. Edit directly in `prompts.csv`. Examples: `"You are a visual analyst."`, `"You are an expert in perception."` |
| 3 | `user_prompt_template` | character | Free text with `{placeholders}` | Prompt content | The actual question. Placeholders: `{direction_a}`, `{direction_b}`, `{test_a_shape}`, `{test_b_shape}`. Filled by `fill_prompt()` at runtime. Edit directly in `prompts.csv`. |
| 4 | `response_format` | character | `"forced_choice"`, `"free_text"`, `"json"` | Prompt design | Tells the parser how to extract the answer. `"forced_choice"` expects a single word, `"free_text"` looks for `ANSWER:` tag. Edit in `prompts.csv`. |
| 5 | `allows_unknown` | logical | `TRUE`, `FALSE` | Prompt design | Whether `"unknown"` is a valid response option presented in the prompt. Edit in `prompts.csv`. |
| 6 | `description` | character | Free text | Metadata | Human-readable label (e.g., `"minimal forced-choice"`, `"expert persona"`). Used in plots. Edit in `prompts.csv`. |

### How to add a new prompt:

Edit `data/prompts.csv` directly:
```csv
prompt_id,system_prompt,user_prompt_template,response_format,allows_unknown,description
4,"You are a measurement robot.","Measure the two {test_a_shape}s. Which is larger: {direction_a}, {direction_b}, or equal?",forced_choice,FALSE,robot persona
```

Then re-run `evaluate_all()`.

## evals.csv — Phase 2: Evaluation

**Generated by:** `evaluate_all()`  
**Used by:** `analyze_results()`  
**Rows:** One per evaluation (trial × prompt × model configuration)

| # | Variable | Type | Values/Range | Role | How to Modify |
|---|----------|------|--------------|------|---------------|
| 1 | `eval_id` | integer | Auto-increment (1, 2, 3, ...) | Primary key | Auto-assigned by `evaluate_all()`. |
| 2 | `trial_id` | integer | FK → `trials.trial_id` | Link (FK) | Links to trials table. Auto-assigned. |
| 3 | `prompt_id` | integer | FK → `prompts.prompt_id` | Link (FK) | Links to prompts table. Auto-assigned. |
| 4 | `provider` | character | `"anthropic"`, `"openai"`, `"google"`, `"github"` | Model config | API provider. Set in `models` list passed to `evaluate_all()`. |
| 5 | `model` | character | Full model ID, e.g., `"gpt-4o-mini"`, `"claude-sonnet-4-20250514"` | Model config | Model identifier. Set in `models` list. |
| 6 | `model_version` | character | Version string, e.g., `"4.5"`, or `NA` | Model config | Optional version label. Set in `models` list. |
| 7 | `temperature` | numeric | `[0, 2]` | Model config | Sampling temperature. Set in `models` list. |
| 8 | `max_tokens` | integer | `> 0` | Model config | Max response tokens. Set in `models` list. |
| 9 | `response_larger` | character | `"equal"`, `"a"`, `"b"`, `"unknown"`, `"parse_error"` | **Result (KEY)** | **Parsed model response**. Compared to `true_larger` in analysis. `"parse_error"` if unparseable. **Read-only** (computed by `parse_response()`). |
| 10 | `response_confidence` | numeric | `[0, 1]` or `NA` | Result | Model's self-reported confidence (if elicited). Currently `NA` (not implemented). |
| 11 | `raw_response` | character | Full text | Result | Complete verbatim model response. **Inspect this to debug parsing issues**. |
| 12 | `latency_ms` | integer | `>= 0` | Performance | Response time in milliseconds. Auto-recorded. |
| 13 | `timestamp` | character | ISO 8601, e.g., `"2026-02-19T23:15:00Z"` | Metadata | When the evaluation was executed. Auto-recorded. |
| 14 | `error` | character | `NA` or error message | Metadata | `NA` if API call succeeded. Error text if failed. |

### How to run evaluations with different settings:

```r
models <- list(
  # Test multiple models
  list(provider = "openai", model = "gpt-4o", temperature = 0, max_tokens = 512),
  list(provider = "google", model = "gemini-2.0-flash", temperature = 0, max_tokens = 512),
  
  # Test temperature effects
  list(provider = "openai", model = "gpt-4o", temperature = 0.5, max_tokens = 512),
  list(provider = "openai", model = "gpt-4o", temperature = 1.0, max_tokens = 512)
)

evaluate_all(trials, prompts, models)
```

Each unique combination (trial × prompt × provider × model × temperature) creates one row in `evals.csv`.

## Derived Analysis Variables — Phase 3

**Computed by:** `analyze_results()`  
**Stored in:** Returned `data` data frame (not saved to CSV)  
**Source:** Join of `trials.csv` + `evals.csv` + `prompts.csv`

| # | Variable | Type | Values | Derived From | Meaning |
|---|----------|------|--------|--------------|---------|
| 1 | `correct` | logical | `TRUE`, `FALSE` | `response_larger == true_larger` | Whether model answered correctly. **Core accuracy metric**. |
| 2 | `illusion_susceptible` | logical | `TRUE`, `FALSE`, `NA` | `tier == 1 & response_larger != "equal"` | On Tier 1 trials (equal sizes): did model answer non-equal? `TRUE` = fooled by illusion. `NA` for non-Tier-1 trials. |
| 3 | `illusion_direction` | character | `"a"`, `"b"`, `NA` | `response_larger` when `illusion_susceptible == TRUE` | On Tier 1 trials where fooled: which side did model pick? Tests directional bias. |
| 4 | `model_label` | character | e.g., `"openai/gpt-4o"` | `paste0(provider, "/", model)` | Human-readable model name for plots. |

### How to compute additional derived variables:

```r
results <- analyze_results()
data <- results$data

# Example: Do surrounds match test shape?
data <- data |>
  mutate(
    surround_a_matches_test = (surround_a_shape == test_a_shape),
    surround_b_matches_test = (surround_b_shape == test_b_shape)
  )

# Example: Surround size ratio
data <- data |>
  mutate(
    surround_a_ratio = surround_a_size / test_a_size,
    surround_b_ratio = surround_b_size / test_b_size
  )
```

## Configuration Variables — config/defaults.R

**Modify these to control what `generate_design()` and `generate_trial()` produce.**

| Variable | Type | Default | Meaning | How to Change |
|----------|------|---------|---------|---------------|
| `SHAPE_POOL` | character vector | `c("circle", "square")` | Shapes available for test and surround stimuli. | Add `"triangle"`, `"hexagon"` when implemented. |
| `TEST_SIZE_RANGE` | numeric vector (length 2) | `c(0.03, 0.08)` | Size range for test stimuli (as proportion of canvas min-dimension). | Increase range for more size variation: `c(0.02, 0.12)`. |
| `SURROUND_SIZE_RANGE` | numeric vector (length 2) | `c(0.02, 0.15)` | Size range for surround stimuli. | Adjust to control illusion strength. |
| `SURROUND_N_RANGE` | integer vector (length 2) | `c(4L, 8L)` | Number of surround stimuli per side. | Increase for more crowding: `c(6L, 12L)`. |
| `SURROUND_DISTANCE_RANGE` | numeric vector (length 2) | `c(0.08, 0.22)` | Distance from test to surround centers (proportion). | Smaller = tighter crowding. |
| `COLOR_POOL` | character vector | `c("#000000", "#4D4D4D", "#999999", "#4682B4", "#B22222")` | Border colors (6-digit hex). | Add more colors: `c(..., "#FF6347", "#00CED1")`. |
| `FILL_POOL` | character vector | `c("#000000", "#4D4D4D", "#999999", "#4682B4", "#B22222", NA)` | Fill colors. Includes `NA` for outline-only. | Add colors or remove `NA` to force fills. |
| `BACKGROUND_POOL` | character vector | `c("#FFFFFF", "#000000")` | Canvas backgrounds (white or black). | Restricted for visibility. Don't add mid-grays. |
| `CANVAS_SIZES` | integer vector | `c(512L, 768L, 1024L)` | Image dimensions (pixels). | Add `2048L` for high-res, or remove `512L` to save space. |
| `ORIENTATION_POOL` | character vector | `c("horizontal", "vertical", "diagonal")` | Spatial layouts. | Remove `"diagonal"` if testing only horizontal/vertical. |
| `FILE_FORMAT_POOL` | character vector | `c("png", "svg", "webp")` | Image formats. | Use `"png"` only for consistency, or test format effects. |
| `FLOAT_TOLERANCE` | numeric | `1e-9` | Tolerance for equality checks in `verify_trial()`. | Don't change unless floating-point issues arise. |
| `DEFAULT_N_PER_TIER` | integer | `50L` | Default trials per tier in `generate_design()`. | Increase for more data: `100L`. |
| `DEFAULT_FILE_FORMAT` | character | `"png"` | Default image format. | Change to `"svg"` for vector graphics. |
| `DEFAULT_CANVAS_WIDTH` | integer | `768L` | Default canvas width. | Match to `CANVAS_SIZES`. |
| `DEFAULT_CANVAS_HEIGHT` | integer | `768L` | Default canvas height. | Match to `CANVAS_SIZES`. |

### Example: Generate larger images with more surrounds

```r
# Edit config/defaults.R:
CANVAS_SIZES <- c(1024L, 2048L)
SURROUND_N_RANGE <- c(8L, 12L)

# Then regenerate:
source("config/defaults.R")
source("R/generate_design.R")
trials <- generate_design(seed = 555, n_per_tier = 30)
render_stimuli(trials)
```

## Model Configuration — evaluate_all()

**Passed as a list to `evaluate_all()`**. Each list element is a named list with these fields:

| Field | Type | Required | Values/Range | Meaning |
|-------|------|----------|--------------|---------|
| `provider` | character | **Yes** | `"anthropic"`, `"openai"`, `"google"`, `"github"` | API provider. |
| `model` | character | **Yes** | Provider-specific model ID (e.g., `"gpt-4o-mini"`, `"claude-sonnet-4-20250514"`, `"gemini-2.0-flash"`) | Model to use. |
| `model_version` | character | No | Version string or `NA` | Optional version label for tracking. |
| `temperature` | numeric | No (default `0`) | `[0, 2]` | Sampling temperature. `0` = deterministic. |
| `max_tokens` | integer | No (default `512`) | `> 0` | Max tokens in response. |

### Example configurations:

```r
# Single model
models <- list(
  list(provider = "openai", model = "gpt-4o-mini", temperature = 0, max_tokens = 512)
)

# Multiple models
models <- list(
  list(provider = "openai", model = "gpt-4o-mini", temperature = 0, max_tokens = 512),
  list(provider = "anthropic", model = "claude-sonnet-4-20250514", temperature = 0, max_tokens = 512),
  list(provider = "google", model = "gemini-2.0-flash", temperature = 0, max_tokens = 512)
)

# Test temperature effects
models <- list(
  list(provider = "openai", model = "gpt-4o", temperature = 0, max_tokens = 512),
  list(provider = "openai", model = "gpt-4o", temperature = 0.3, max_tokens = 512),
  list(provider = "openai", model = "gpt-4o", temperature = 0.7, max_tokens = 512),
  list(provider = "openai", model = "gpt-4o", temperature = 1.0, max_tokens = 512)
)

evaluate_all(trials, prompts, models)
```

## Summary: What to Edit for Common Tasks

| Task | What to Edit | Where |
|------|--------------|-------|
| **Change stimulus shapes** | `SHAPE_POOL` | `config/defaults.R` |
| **Make illusion stronger/weaker** | `SURROUND_SIZE_RANGE` | `config/defaults.R` |
| **Add more trials** | `n_per_tier` argument | `generate_design(n_per_tier = 100)` |
| **Test only horizontal layouts** | Filter trials or set `orientation` | `trials |> filter(orientation == "horizontal")` |
| **Add a new prompt** | Add row | `data/prompts.csv` |
| **Test a new model** | Add to `models` list | `evaluate_all(trials, prompts, models)` |
| **Change image resolution** | `CANVAS_SIZES` | `config/defaults.R` |
| **Exclude Tier 0 trials** | Filter trials | `trials |> filter(tier > 0)` |
| **Test temperature effects** | Add multiple model configs with different `temperature` | `evaluate_all()` |
| **Compute custom metrics** | Add derived columns | In `analyze_results()` or post-hoc on `results$data` |

## Variable Naming Conventions

- **Columns with `_a` / `_b` suffix:** Refer to stimulus positions A (left/top/upper-left) and B (right/bottom/lower-right).
- **Columns ending in `_ratio`:** Decimal proportions (0–1 scale).
- **Columns ending in `_n`:** Counts (integers).
- **Columns ending in `_id`:** Primary or foreign keys (integers).
- **All color values:** 6-digit hex codes (e.g., `"#4682B4"`) or `NA`.
- **All paths:** Relative to project root (e.g., `"images/1_b_t3.png"`).
